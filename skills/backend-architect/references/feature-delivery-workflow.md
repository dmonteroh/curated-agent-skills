# Feature Delivery Workflow (Backend)

Use this to coordinate backend feature delivery end-to-end.

## Phases

1. **Discovery**: clarify requirements, acceptance criteria, risks.
2. **Architecture**: define service boundaries, API contracts, data model.
3. **Implementation**: build APIs, resilience, and observability.
4. **Testing**: unit + integration + security checks.
5. **Release**: rollout strategy, feature flags, rollback plan.

## Outputs

- API contract and data model.
- Threats + mitigations.
- Verification plan (tests + monitoring).

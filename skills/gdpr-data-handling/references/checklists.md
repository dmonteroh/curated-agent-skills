# GDPR Implementation Checklist

Use these checklists to validate coverage during reviews.

## Legal Basis

- Document lawful basis for each processing purpose.
- Capture consent evidence where consent is required.
- Record legitimate interest assessments when used.

## Transparency

- Privacy notice includes purposes, retention, and rights.
- Data flows are documented and accessible to stakeholders.

## Data Subject Rights

- Access, erasure, rectification, portability workflows exist.
- Response timelines tracked with escalation for delays.
- Evidence of completion is retained.

## Security and Access

- Encryption at rest/in transit where appropriate.
- Least-privilege access with periodic review.
- Logging avoids unnecessary personal data exposure.

## Breach Response

- Incident triage process and contact list.
- Authority notification decision logged within 72 hours.
- Data subject notification criteria documented.

## Documentation

- Records of processing activities (Art. 30) maintained.
- Data processing agreements with vendors documented.

## Best Practices: Do

- Minimize data collection and retention.
- Provide clear opt-in controls for optional processing.
- Test DSAR workflows with sample requests.
